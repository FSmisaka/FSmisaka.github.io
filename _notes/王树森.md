# 绪论

## 转化流程

曝光 → 点击 → 点赞/收藏/转发/{滑动到底 → 评论}

## 评价指标

### 消费指标

- 指标
    
    点击率 = 点击 / 曝光  
    点赞率 = 点赞 / 点击  
    收藏率 = 收藏 / 点击  
    转发率 = 转发 / 点击  
    阅读完成率 = 滑动到底 / 点击 * f(笔记长度)  
    
- 这些比率实际上都和转化流程有关：比如点击率就是$\frac{点击}{点击的上游流程}$。唯一的例外是阅读完成率，其公式中的笔记长度是为了对长笔记更公平

### 北极星指标

1. 用户规模  
日活 DAU、月活 MAU
2. 消费  
人均时长、人均阅读量
3. 发布  
发布渗透率、人均发布量

## 实验流程

离线实验（历史数据）→ AB test（小流量线上实验） → 上线

### AB test

- 随机分桶：每个桶使用不一样的策略，每个桶中的被试是同质的  
- 分层实验：同层互斥、不同层正交（每个层都随机给用户分桶）  
—— 比如分层分为召回层、粗排层、精排层、重排层，并且我们关注的是在召回层做的 ab test。虽然a组和b组的同一用户很可能会在粗排等其它层中进到不同的ab test组，导致在很细的粒度看，似乎你没有办法分清楚最后结果的显著到底是因为召回层的差异带来的还是其它层的差异带来的。但是，从整体的角度，由于每个层都随机分桶，a组和b组的用户在其它层有着相同的“分桶分布”，因此整体来看a组和b组的用户还是同质的，即最后结果的显著还是说明是召回层的差异带来的。  
—— 同层一般是不正交的，策略会互相影响，所以用户在一个层中一般只能进一个桶。
- Holdout桶：计算部门整体的业务指标收益  
—— 部门有很多人，每几个人共同负责一种算法优化。不能简单把这些算法优化的指标提升相加。那么怎么判定整体部门的收益呢，只好分出一部分用户进Holdout桶，这些人不会进任何ab test。
- 反转桶：在新的推全层保留一个小的反转桶，使用旧策略，目的是为了观察一些滞后的效应

## 协同过滤

### ItemCF

- 用户 $u$ → 喜欢的物品 $i$ → 相似物品 $j$
    
    $\sum_{i \in {u喜欢的物品}} \mathrm{like}(i)·\mathrm{sim}(i,j)$
    
- 物品相似度：全部用户的评分向量的余弦相似度
    
    $cos(S_i,S_j) \ = \ \frac{S_i ·S_j}{|S_i|·|S_j|}$
    $S_i$：物品 $i$ 的用户评分
    
- 离线构建 2 个索引：用户→物品，物品→物品，这样线上召回就快了

### Swing 相似度

- 优化：在 ItemCF 判别物料相似度时，尽管使用的全量用户的评分向量，真正起作用的其实是同时交互过的用户。如果这些同时交互物料的用户恰好是一个小圈子的，就会导致把不相似的物料也变得很相似。
- $\mathrm{sim}(i_1,i_2) \ = \ \sum_{u_1 \in V}\sum_{u_2 \in V}\frac{1}{\alpha+\mathrm{overlap}(u_1,u_2)}$
$V$：同时交互过 $i_1$ 和 $i_2$ 的用户
$\mathrm{overlap}$：$u_1$ 和 $u_2$ 的重合度

### UserCF

- 用户 $u$  → 相似的用户 $v$ → 喜欢的物品 $j$
- 用户相似度：用户给全量物品的打分向量
    
    $cos(S_u,S_v) \ = \ \frac{S_u·S_v}{|S_u|·|S_v|}$  
    另外，热门产品对于用户相似度计算是没有任何帮助的，只有冷门的物料才能帮助计算用户相似度，因此有一些相应的改进。
    

## 向量召回

### 离散特征处理：向量化

映射为序号 → 向量化

- 高维稀疏向量：One-hot 独热编码
    
    e.g. 国籍特征：CN、US、JP、KR、SG  
    序号字典 {CN: 0, US: 1, JP: 2, …}  
    One-hot(CN) = [1, 0, 0, …, 0]  
    One-hot(US) = [0, 1, 0, …, 0]  
    One-hot(JP) = [0, 0, 1, …, 0]  
    
    局限：类别数量巨大时，维度过于大，向量过于稀疏  
    
- 低维稠密向量：Embedding
    
### 矩阵补充（FunkSVD）

$\min_{\mathrm{emb_u},\mathrm{emd}_i}\sum_{\{u,i,y\}\in \Omega}{(<\mathrm{emd_u}(u), \mathrm{emb_i}(i)> \ - \ y)^2}$ 

- 实践中不会用，不 work
    1. 没有结合用户和物品的属性，只用 id
    2. 负样本选择不好
    3. 训练有问题：MSE损失不如交叉熵；内积不如余弦相似度

### 双塔模型

- 双塔模型是后期融合的模型
    
    双塔不会把用户特征和物品特征拼在一起送进神经网络，而是分开送进神经网络，后期再计算相似度。这样做是因为召回阶段的候选数据非常庞大。这样做可以提前离线计算物品embedding，大大减小计算量。

![截屏2026-04-12 12.02.46.png](/images/notes/王树森/%E6%88%AA%E5%B1%8F2026-04-12_12.02.46.png)

- Pointwise 训练：用户会不会喜欢物品？
    
    **对比损失**
    $\min_{\Theta}y·\mathrm{D}^2(\mathrm{emb_u}(u;\Theta),\mathrm{emb_i}(i,\Theta)) \ + \ (1-y)·\max^2(0, \mathrm{margin}-\mathrm{D}(\mathrm{emb_u}(u;\Theta),\mathrm{emb_i}(i,\Theta)))$
    
    如果是正样本，鼓励距离接近；如果是负样本，鼓励距离大到超过margin
    正负样本比例，1:2；1:3
    
- Pairwise 训练：对于用户来说，物品i和物品j哪个更好？
    
    **Triplet Hinge Loss**
    $\max_{\Theta}(0,\mathrm{margin}-\{\mathrm{D}[\mathrm{emb_u}(u;\Theta),\mathrm{emb_i}(i_{negative};\Theta)]-\mathrm{D}[\mathrm{emb_u}(u;\Theta)-\mathrm{emb_i}(i_{positive};\Theta)]\})$
    
    鼓励负样本距离超过正样本距离至少margin
    
    **Triplet Logistic Loss**
     $\min_{\Theta}-log(\sigma(D[\mathrm{emb_u}(u;\Theta),\mathrm{emb_i}(i_{negative};\Theta)]-D[\mathrm{emb_u}(u;\Theta),\mathrm{emb_i}(i_{positive;\Theta})]))$
    
    鼓励扩大负样本距离超过正样本距离的概率
    
- Listwise 训练：对于用户来说，下面的物品list应该怎么排序？
    
    **Softmax激活函数** & **Cross Entropy Loss**
    $\min_{\Theta}-log\frac{\exp{\mathrm{sim}[\mathrm{emb_u}(u;\Theta),\mathrm{emb_i}(i_{positive};\Theta)]}}{\sum_{i \in 物品样本}{\exp{\mathrm{sim}[\mathrm{emb_u}(u;\Theta),\mathrm{emb_i}(i;\Theta)]}}}$
    
- 正样本的选择
    
    正样本：用户曝光+点击的物品
    
    如果不做任何处理，正样本大多数都是热门物品，这样对冷门物品不公平
    解决方案：过采样冷门物品、降采样热门物品
    过采样（up-sampling）：一个物品采样多次
    降采样（down-sampling）：抛弃一些物品
    
- 负样本的选择
    
    负样本：在整个推荐流程中被淘汰的物品，包括没有被召回、召回但没有被曝光、曝光但没有被点击……
    
    1. 简单负样本（全体物品）
    非均匀抽样（抽样概率 $\propto 点击次数^{0.75}$）
    2. 简单负样本（Batch内物品）
    其它用户的点击物品成为该用户的负样本
    —— 纠偏：问题是，此时负样本的抽样概率 $\propto 点击次数$，导致热门物品出现概率偏大，模型对热门物品的打压过重，因此需要纠偏。纠偏方式是：训练时 $cos(u,i) \rightarrow cos(u,i)-log(抽样概率)$，召回时还是用原本的余弦相似度
    3. 困难负样本（召回但被粗排淘汰，比较困难，模型容易把它们判断为正样本）
    4. 困难负样本（精排被淘汰，非常困难，模型很容易把它们判断为正样本）
    5. 错误的召回负样本：曝光但没有被点击（适合作为排序负样本）
    召回模型的任务：分清楚用户 感兴趣 vs 不感兴趣
    排序模型的任务：分清楚用户 比较感兴趣 vs 非常感兴趣
    
    实际做法：50% 简单负样本 + 50% 困难负样本 
    
- 自监督学习
    
    解决问题：高点击物品的表征学得好，长尾物品学得不好。自监督学习就是对长尾物品做数据增强 data augmentation
    
    整体逻辑：
    $i_1, i_2\xrightarrow{特征变换_1}i_1',i_2'$； $i_1, i_2\xrightarrow{特征变换_2}i_1'',i_2''$
    训练目标：min 不同物品相同变换后的相似度；max 相同物品不同变换后的相似度
    
    特征变换 - Random Mask：随机 Mask 一些特征
    特征变换 - Dropout：随机丢弃多值特征中50%的值
    特征变换 - Complementary：分两组，a组用一部分特征，b组用这些特征的补集
    特征变换 - Mask 一组关联特征，计算公式如下：
     $MI(U,V) = \sum_{u \in U}\sum_{v \in V}p(u,v)log\frac{p(u,v)}{p(u)·p(v)}$
    
    从全体物品中随机抽样 $m$ 个物品，并做两类特征变换得到
     $b_1',b_2',\dots,b_m'$ 和 $b_1'',b_2'',\dots,b_m''$
     损失函数： $\min_{embedding}L_{self} \ = \ \frac{1}{m}\sum_{i=1}^{m}-log\frac{\exp{cos(b_i',b_i'')}}{\sum_{j=1}^{m}{\exp{cos(b_i',b_j'')}}}$
    
    整体的损失函数为 $\min_{embedding}\mathrm{infoNCE}+\alpha L_{self}$
    

### 离线存储

物品向量：离线计算并存储、建立索引
用户向量：线上计算

全量更新（天级别）：
—— 更新用户塔全量神经网络、更新物品向量。
—— 用 random shuffle 一天的数据，训练 1 epoch
增量更新（小时级别）：
—— 只更新用户塔 embedding 层。目的是维护最新的用户兴趣
—— 用当天从早到晚的数据，训练 1 epoch

—— 只做增量则效果不好，全量更新中的 random shuffle 很重要

### 线上召回：查找最相似的向量

- 最近邻查找
    
    用户 u → 在哈希表中找到 u 的 emb(u)
    遍历物品表，找到最大的 k 个 <emb(u), emb(i)>
    
- 近似最近邻查找 ANN
    - 最近邻标准
        
        min 欧氏距离  
        max 内积  
        max 余弦相似度  
        
    - IVF（Inverted File Index）
        
        划分向量空间（比如用聚类），每一个划分用一个单位向量表示
        
        ![截屏2026-04-12 10.48.56.png](/images/notes/王树森/%E6%88%AA%E5%B1%8F2026-04-12_10.48.56.png)
        
        查找时，先和单位向量比较找到划分，再和划分中的向量一一做比较
        
        ![截屏2026-04-12 10.50.46.png](/images/notes/王树森/%E6%88%AA%E5%B1%8F2026-04-12_10.50.46.png)
        
    
    工业界常用 Faiss，这是一个 ANN 算法工具箱
    
- 为什么不用快速选择/堆排序？LeetCode上的找 k 个最大数不就这么做的吗？
    
    这些方法的前提是相似度已经被计算出来了。而计算相似度需要遍历所有物品。
    
    最近邻查找是筛选 candidate，实际只会和这些 candidate 做相似度的计算
    

## 其它召回通道

- Deep Retrieval 召回
    
    ![截屏2026-04-13 21.29.40.png](/images/notes/王树森/%E6%88%AA%E5%B1%8F2026-04-13_21.29.40.png)
    
    把物品表征为路径，线上查找用户最匹配的路径。  
    —— 我的理解是，路径（即物品）上的节点是一些小的内容要素，一个物品是包含多个内容要素的，比如一条视频可能包括是否包括人、性别要素、内容类别等等。
    
    用户最感兴趣的物品可能就是：包括人、男性、足球、…  
     $用户最感兴趣的物品 = \Pi_{要素维度}用户在要素维度中最感兴趣的要素$
    
    - 索引
        
        召回时用到：path → List<item>  
        训练时用到：
        
    - 预估模型
        
        $p(路径|走过的节点;\mathbf{x})$ 代表给定用户特征 $\mathbf{x}$ 和已经走过的节点，用户对路径（包含走过的节点和当前要走的节点）的兴趣。  
        —— 我理解就是，给定用户特征和用户对包含了前 $i-1$ 个要素的物品的兴趣，预估用户对包含了 $i$ 个要素的物品的兴趣。  
        
        $p(a_1,a_2,\dots,a_l|\mathbf{x}) \ = \ p_1(a_1|\mathbf{x})\times p_2(a_2|a_1;\mathbf{x}) \times \dots \times p_l(a_l|a_1,a_2,\dots,a_{l-1};\mathbf{x})$
        
    - 线上召回
        
        forward：依次选择要素维度（当前步的节点集）中用户最感兴趣的内容要素（节点）

# 排序

## 排序模型的特征

### 特征

- 用户画像：创立用户时就会有的信息
    
    User ID、人口属性、账号信息、用户基本兴趣、…
    
- 物品画像：发布物品时就会有的信息
    
    Item ID、时空信息、笔记的字数、...
    
- 统计特征
    
    用户统计特征：最近一定天数用户的曝光、点击、点赞、收藏、…
    —— 按物料分桶，比如图文/视频分2个桶、按类目分桶、…
    物料统计特征：最近一定天数物料的曝光、点击、点赞、收藏、…
    —— 按用户分桶，比如按性别、年龄分桶、…
    
- 场景特征
    
    时空信息、是否周末/节假日、操作系统

## 粗排

### 三塔模型（用户塔，物品塔，交叉塔）

![介于前期融合（多目标、MMoE）和后期融合（双塔）之间](/images/notes/王树森/%E6%88%AA%E5%B1%8F2026-04-15_15.09.06.png)

介于前期融合（多目标、MMoE）和后期融合（双塔）之间

如果有 $n$ 个物品，模型上层需要做 $n$ 次推理，贡献了粗排的大部分计算量

交叉塔计算实时变化的统计特征，对 $n$ 个物品做 $n$ 次推理，塔必须得比较小；
物品塔可以离线计算，缓存结果，只有遇到新物品时才需要推理，计算量不大；
用户塔只有 $1$ 个用户，计算量不大。

        原本的预估：$\hat{y} \ = \ \frac{n_+}{n_+ + \alpha·n_-}$
        
        真实应该是：$y' \ = \ \frac{n_+}{n_+ + n_-}$

### FM 因子分解机 Factorized Machine

线性模型 + 二阶交叉特征：（问题是系数矩阵 $U$ 太复杂）
 $p = b+\sum_{i=1}^{d}w_ix_i+\sum_{i=1}^d\sum_{j=i+1}^du_{ij}x_ix_j$

FM：（矩阵分解 $U_{d\times d} = V_{d \times k}V_{k\times d}^T$）
 $p = b+\sum_{i=1}^{d}w_ix_i+\sum_{i=1}^d\sum_{j=i+1}^d(v_iv_j^T)x_ix_j$

FM 其实是线性模型（线性回归、逻辑回归）的升级，在线性项后面加入二阶交叉特征；早起推荐系统使用逻辑回归预测点击，而在逻辑回归后加入 FM 效果很好

## 行为序列

### 用户历史行为序列建模

LastN 特征：最近 $n$ 个交互物品的 各种 embedding 的平均

各种 embedding：不会只用 ID embedding，这种效果不那么好
取平均：最简单但效果一般，近期的论文有优化

### DIN

会取 $n$ 个物品的加权平均，不再仅仅是简单的平均
权重就是当前在打分的物品与 LastN 物品的相似度

![截屏2026-04-16 10.41.18.png](/images/notes/王树森/截屏2026-04-16_10.41.18.png)

DIN 的本质就是注意力机制

- 简单平均 vs 注意力机制
    
    都可以用在精排模型中，注意力机制效果更好
    
    只有简单平均能被用在召回双塔、粗排三塔中
    —— 简单平均得到的 LastN 特征单纯是用户特征；注意力机制得到的 LastN 特征实则是用户-物品交叉特征
    

### SIM

DIN 模型的缺点：LastN 不能太大否则无法计算；模型关注短期兴趣，遗忘长期兴趣

改进思路：先用简单的方式，快速排除 LastN 中与当前打分物品无关的物品  
这种做法可以直接让 $n$ 从 200 到 几k  

查找（排除）的做法：

1. 基于规则，例如要求类目相同  
2. 基于向量相似度，先学习 embedding，再做最近邻查找  

SIM 建模长序列，时间信息也很关键，需要引入（DIN 则是没有引入的，LastN 基本都是很近的交互）。实践中会先分桶再向量化，最后与 LastN 向量做 Concatenation

## 多样性算法

### MMR（Maximal Marginal Relevance）

$\operatorname*{arg\,max}_{i \notin \mathrm{selected}}MR_i$  
$MR_i \ = \ \theta·\mathrm{reward}_i \ + \ (1-\theta)·\max_{j \in \mathrm{selected}}\mathrm{sim}(i,j)$  
$\mathrm{reward}$：精排的分数  

每轮都选择分数大，与已选中物品相似度小的物品

- 滑动窗口优化
    
    当 $\mathrm{selected}$  集非常打时，很可能没有被选择的物品与已选物品均非常相似，这个时候 $MR$ 中的第二项不再起作用（因为大伙的分数都一样），就可能连着好几个物品非常相似。
    
    解决方法是把 $\mathrm{selected}$ 改为滑动窗口，只把未被选择的物品与刚刚选择的 $w$ 个物品比较相似度；尽管用户可能刷了一会r后又会刷到最开始刷过的类似内容，但是不容易出现连着好几个特别相似了。
    
    $MR_i \ = \ \theta·\mathrm{reward}_i \ + \ (1-\theta)·\max_{j \in \mathrm{window}}\mathrm{sim}(i,j)$
    

### 行列式点过程 DPP

- 超平行体
    
    $K$ 维超平行体：由 $K$ 个 $d$ （$K \leq d$）维线性无关基向量 $\mathbf{v}_1,\dots,\mathbf{v}_K \in \mathbb{R}^d$ 张成的空间 $\mathcal{P}(\mathbf{v}_1,\dots,\mathbf{v}_K)=\{ \alpha_1\mathbf{v}_1+\dots+\alpha_K\mathbf{v}_K \ | \ 0 \leq \alpha_1,\dots,\alpha_K\leq1 \}$
    
    超平行体的体积：衡量基向量的多样性  
    —— 体积大，多样性好，极端情况基向量互相正交，体积 = 1  
    —— 体积小，多样性差，极端情况基向量线性相关，体积 = 0  
    
    把基向量拼成矩阵 $\mathbf{V} = [\mathbf{v}_1,\dots,\mathbf{v}_K]$，那么  
     $\det(\mathbf{V}^T\mathbf{V}) = \mathrm{volume}^2(\mathcal{P}(\mathbf{V}))$  
    → 可以直接用行列式来衡量向量的多样性  
    
- 重排多样性
    
    输入：1. 每个物品的精排打分 $\mathrm{reward}$；2. 每个物品的 embedding  
    目标：1.  $\max \sum_{i \in \mathrm{selected}}\mathrm{reward}_i$；2. $\max \mathrm{volume}(\mathcal{P}(\mathrm{selected}))$  
    
    DPP：$\operatorname*{arg\,max}_{\mathrm{selected}}log\det(\mathbf{V}_{\mathrm{selected}}^T\mathbf{V}_{\mathrm{selected}})$  
    
    推荐系统：$\operatorname*{arg\,max}_{\mathrm{selected}}\theta·\sum_{i \in \mathrm{selected}}\mathrm{reward}_i \\ + \ (1-\theta)·log\det(\mathbf{V}_{\mathrm{selected}}^T\mathbf{V}_{\mathrm{selected}})$
    
- 贪心求解（与 MRR 类似的策略）
    
    $\operatorname*{arg\,max}_{i} \theta·\mathrm{reward}_i \ + \ (1-\theta)·log\det(\mathbf{V}_{\mathrm{selected} \ \cup \ i}^T\mathbf{V}_{\mathrm{selected} \ \cup \ i})$
    
- 滑动窗口优化
    
    $\operatorname*{arg\,max}_{i} \theta·\mathrm{reward}_i \ + \ (1-\theta)·log\det(\mathbf{V}_{\mathrm{window} \ \cup \ i}^T\mathbf{V}_{\mathrm{window} \ \cup \ i})$
    

### 业务规则约束（XHS）

1. 最多连续出现 k 个某种类型的笔记
2. k 篇笔记中最多出现 1 篇某种笔记
3. 前 t 篇笔记至多出现 k 篇某种笔记

MMR/DPP + 业务规则：  
每一轮在选择物品时会先用规则过滤不合法的物品，然后再用算法选出物品

1. ID Embedding ← Default Embedding，至少强于随机初始化/全 0 初始化
2. 使用相似高曝笔记的 Embedding

### 类目、关键词召回

用户 → 持续维护的用户画像 → 类目/关键词 $\xrightarrow{类目/关键词\rightarrow笔记 \ 倒排表}$ 笔记  

倒排表是指对时间倒排，因此这个召回通道能够召回新笔记

### 物品聚类召回

召回：用户 → LastN → 找到最相似的聚类 → 获取聚类中的笔记  
新物品：找到最相似的聚类 → 更新聚类中的笔记（关于时间倒排）  

### Look-Alike 人群扩散

思想：新笔记 → 最初交互过笔记的用户 embedding $\xrightarrow{embedding的平均}$ 新笔记的特征向量 → 根据这个特征，扩散初始用户

召回：用户 embedding $\xrightarrow{新笔记特征向量数据库；最近邻查找}$ 相近的几篇新笔记
